# Project Summary
El Crunchae is a premium eCommerce platform specializing in freeze-dried fruits, vegetables, and ready meals. It offers a seamless shopping experience with integrated payment processing, customizable shipping options, and an intuitive admin panel for inventory management. Recent updates include a dual-mode authentication system that allows the application to function in both development (using local storage) and production (using PHP/MySQL).

# Project Module Description
- **Backend**: Utilizes PHP/MySQL for data management, user authentication, payment processing, order management, and email notifications.
- **Frontend**: Built with React, offering a responsive user interface for browsing products, managing the cart, and tracking orders. Key features include lazy loading for images, advanced search capabilities, a dynamic testimonials carousel, and a new admin feature for site settings.
- **Admin Panel**: Provides tools for product management with access controls for adding, updating, and removing products, alongside new site settings for adjusting the hero section.

# File Description Inventory
- **frontend/api/**: PHP files for API operations including configuration, authentication, product management, reviews, and orders.
- **frontend/src/lib/**: Core logic for data operations and authentication using the PHP/MySQL backend.
- **frontend/src/pages/**: Contains route pages for product management and user authentication, including updated About and Business pages.
- **frontend/src/components/**: Reusable UI components, including the updated navigation bar, footer, TestimonialsCarousel, and LoginModal.
- **frontend/src/contexts/**: Manages authentication state across the application.
- **frontend/src/lib/siteSettings.ts**: Manages site settings stored in localStorage for admin-configurable options.
- **frontend/src/lib/api.ts**: Implements a dual-mode authentication system to handle both development and production environments seamlessly.
- **frontend/vite.config.ts**: Configuration file for building the frontend application.
- **frontend/public/assets/**: Contains all local images used in the application, including the new hero background image.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Framer Motion, Shadcn-ui
- **Backend**: PHP, MySQL
- **Database**: MySQL
- **Deployment**: Hostinger, Vercel, Netlify

# Usage
1. **Install Dependencies**: Navigate to the frontend directory and use the package manager to install dependencies.
2. **Build the Project**: Execute the build command to compile the frontend application.
3. **Deploy**: Upload the built `dist/` folder to the desired hosting provider. Ensure the PHP API files are uploaded to the correct directory on Hostinger.
